Part I — Member Guide

1. Sign-In and First Access

Shalom uses OTP authentication instead of passwords. A member signs in using an Indian mobile number already registered in the church database. The app automatically prefixes the number with +91 and sends a one-time code via SMS.

1

Open the app

Go to shalomapp.in or open the installed PWA from your device home screen.

2

Enter the registered phone number

Type the 10-digit number. The app displays the +91 prefix separately.

3

Tap Send OTP

The code arrives via SMS. If expired, request a fresh OTP — never reuse an older code.

4

Verify the code

Enter the 6-digit OTP and tap Verify OTP to enter the workspace assigned to your role.

Important: if the phone number is not already linked to a member record, access will fail. The church admin must pre-register the member account first.

3. Profile, Family, and Special Dates

The profile area combines personal details, membership identifiers, family relationships, and special dates. Keep this record current so the church can contact you and run accurate exports.

1

Update personal details

Edit full name, alternate phone, address, gender, date of birth, occupation, age, and confirmation status.

2

Upload or refresh the photo

Upload a profile image and use the crop flow before saving.

3

Add family links

Search within Registered church members, choose the relationship, and submit the request for admin approval.

4

Maintain special dates

Add birthdays and anniversaries so the church can track greetings and export records.

Family Management Rules

  • Family links use existing church member records, No member can be linked twice.
  • Every request is visible to admins for approval or rejection.
  • Linked family dues appear on the dashboard for combined payment.

Phone Number Change

  • Changing the primary phone requires OTP verification of the new number.
  • Complete the OTP flow before leaving the profile page.
  • If verification fails, keep the old number and retry later.

Operates admin tools for their church: members, subscriptions, payments, communications, exports, and approval queues.

7. Member and Church Operations

1

Search the target record

Use name, email, phone, or membership ID within your church context.

2

Inspect profile and verification status

Review pending, verified, rejected, or suspended state before making changes.

3

Edit controlled fields

Update contact data, address, DOB, occupation, membership ID, and subscription amount where policy permits.

4

Use delete actions carefully

Deletion tools surface impact previews for cascading relationships before irreversible operations are confirmed.

9. FAQ, Troubleshooting, and Good Practice

What should a member do if OTP verification fails?

Request a fresh OTP and avoid reusing an older code. If the number is correct but access still fails, the church admin should verify that the phone number is linked to the correct member account.

Why is a family member not appearing for selection?

The person must already exist as a member record in the same church context, and the request may still require admin approval before the link is active.

What is the safest order for admin data changes?

Search first, validate church and member context second, edit carefully third, and only then use delete or restore tools if absolutely required.

How should receipts and reports be handled?

Use downloaded receipts for member confirmation and CSV exports for reconciliation, audit support, and financial reporting.

How can admins handle pending approval queues?

Check Family Requests, Cancellation Requests, and Account Deletion Requests tabs daily. Unreviewed items stay pending and may delay member actions.

Operational Best Practices

  • Keep member phone numbers current before asking them to sign in.
  • Use approval tabs daily to clear pending family, cancellation, and deletion requests.
  • Review payment gateway settings before high-volume payment campaigns.
  • Export reports on a schedule so financial review does not depend on ad-hoc manual pulls.
  • Use push and SMS targeting carefully to avoid irrelevant broadcasts.
  • Always preview the deletion impact summary before confirming any delete operation.
